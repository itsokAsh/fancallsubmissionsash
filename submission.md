# 📝 Submission Guidelines

Follow these steps to submit your hackathon project:

## 🚀 Step-by-Step Submission Process

### 1. Fork This Repository
- Click the "Fork" button at the top right of this repository
- This creates your own copy of the repository where you can make changes

### 2. Clone Your Forked Repository
```bash
git clone https://github.com/YOUR_USERNAME/opencode-gittea.git
cd opencode-gittea
```

### 3. Create a Folder with Your Name
- Create a new folder in the repository root
- Name it using your full name (e.g., `john-doe` or `jane-smith`)
- This folder will contain all your project files

```bash
mkdir your-name
cd your-name
```

### 4. Add Your Project Files
- Place all your project files inside your named folder
- Include:
  - Your code files
  - Documentation (README.md explaining your approach)
  - Any additional files relevant to your submission

### 5. Commit and Push Your Changes
```bash
git add .
git commit -m "Add submission: [Your Name] - Recommendation System"
git push origin main
```

### 6. Raise a Pull Request (PR)
- Go to the original repository on GitHub
- Click on "Pull Requests" tab
- Click "New Pull Request"
- Select your forked repository and branch
- Add a descriptive title and description of your submission
- Submit the pull request

## 📋 PR Requirements

Your Pull Request should include:
- ✅ Clear title with your name
- ✅ Description of your recommendation system approach
- ✅ Key features and logic implemented
- ✅ Any assumptions or trade-offs made

## ⚠️ Important Notes

- **Do not modify files outside your folder**
- **Do not delete or modify existing repository files**
- **Keep your folder structure organized**
- **Ensure your code is well-documented**

## ❓ Need Help?

If you encounter any issues during submission, please reach out to the hackathon organizers.

---

*Good luck with your submission! 🚀*
